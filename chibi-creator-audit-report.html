Section 03
🔴 Critical Issues — Security

These are expected limitations of a static-only frontend. Acceptable for a demo/prototype — must be resolved before any production deployment.

C1
Password "hashed" with btoa() — trivially reversible
Both register() and login() use btoa(password) which is Base64 encoding, not hashing. Anyone with read access to the users table can decode all passwords with atob() instantly.
register() line 341 login() line 321 Risk: credential exposure
C2
Full user object stored in localStorage (XSS-readable)
The complete user record including email and plan is stored as a plain JSON string in localStorage under the key chibi_user. Any XSS vulnerability can extract this data.
localStorage.setItem('chibi_user', ...) Risk: user data exposure via XSS
C3
OpenAI API key stored unencrypted in localStorage
The user's OpenAI secret key (sk-proj-…) is stored in plain text in localStorage under chibi_openai_key. An XSS attack or malicious browser extension can steal it and run up billing charges.
setApiKey() line 36 Risk: OpenAI billing key theft
Section 04
🟠 High Severity — Functional Bugs
H1
Photo-to-Chibi ignores the uploaded image
photoToChibiWithOpenAI builds a text prompt describing identity preservation but passes only text to DALL-E 3, which has no image input capability. The base64 photo data is accepted by the function but never sent to OpenAI. The Photo tab produces essentially the same output as the Text tab.
Lines 531–555 Impact: Photo tab is non-functional
H2
HD Download always downloads the same 1024×1024 image
The hd flag sets quality: 'hd' in the DALL-E 3 request but the size parameter stays at CHIBI_CONFIG.IMAGE_SIZE which is '1024x1024'. Pro users clicking "HD Download" get the same resolution as the standard download — misleading.
generateWithOpenAI lines 500–507 Impact: HD button misleads Pro users
H3
Quota increment race condition
incrementUsage() reads quotaInfo.count from React state and writes count + 1. If two generations are triggered in rapid succession (e.g., double-click), both read the same stale count and each writes count + 1, resulting in an undercounted quota total.
Line 1238 Impact: Quota bypassed via rapid clicking
H4
Gallery sort ascending — newest items appear last
loadPublicGallery passes sort: 'created_at' to the Table API which sorts ascending by default. New user saves appear at the bottom of the 50-item gallery, buried beneath sample items that have hard-coded high like counts.
Line 1739 Impact: New chibis never visible in gallery
H5
Active nav highlight never fires for "How it Works"
The section tracker in App() observes IDs ['hero','studio','gallery','ux','architecture','pricing']. When the ux section is visible, setActiveSection('ux') is called correctly — but the ux link in the navbar matches id:'ux', so this actually should work. However the 'architecture' section ID is observed but the architecture section has id='architecture' while no nav link points to it, meaning the nav highlight gets stuck on 'architecture' while the user scrolls through pricing.
App() lines 2354–2361 Navbar navLinks line 907 Impact: Nav highlight wrong at bottom of page
Section 05
🟡 Medium Severity — UX Problems
M1
Post-generation action bar too crowded (9 buttons)
The action toolbar renders: PNG, HD, Save, Share, Copy, 📄 (meta export), Zoom, 🔄 (regenerate), and "Add Key" — nine buttons on one row. On mobile (<480px) these wrap to 3–4 lines. The 📄 meta export icon is cryptic to regular users.
Lines 1685–1694 Impact: Overwhelming on mobile
M2
UX Section falsely claims "GPU-accelerated Stable Diffusion XL"
The "Instant Generation" feature card reads: "Results in under 10 seconds with our optimized AI pipeline backed by OpenAI and GPU-accelerated Stable Diffusion XL." The app exclusively uses DALL-E 3. SDXL is not integrated.
UXSection line 1962 Impact: False advertising
M3
Architecture diagram shows unimplemented stack as active
The Architecture section displays OpenAI gpt-image-1, Stable Diffusion XL, Real-ESRGAN Upscaler, PostgreSQL + Prisma, Redis Cache, Cloudflare R2, BullMQ, Vercel Edge — none of which are connected. The actual stack is DALL-E 3 + a static Table API.
ArchSection lines 1993–1999 Impact: Misleads technical evaluators
M4
Footer brand description messaging inconsistency
Footer says: "Transform anyone into an adorable anime character." Hero says: "premium chibi portrait — social-ready, identity-preserving." These are different positioning statements creating brand confusion.
Footer line 2302 Impact: Inconsistent brand voice
M5
Quota jump from 3 (guest) to 5 (free account) unexplained
Guest users see "3 free/day" in the quota bar, but the sign-in CTA says "Sign in for 5/day." The jump from 3 to 5 is never explained. Users may not understand why signing up gives more, or may feel the guest limit is artificially low.
Lines 1424–1436 GUEST_DAILY_LIMIT = 3, FREE plan = 5
M6
handleExportMeta leaks blob URL (memory)
handleExportMeta creates a Blob and URL.createObjectURL(blob) but never calls URL.revokeObjectURL() after the click. Each metadata export leaks a blob reference in memory for the lifetime of the page.
handleExportMeta line 1380 Impact: Minor memory leak
M7
Demo mode "No API key" banner visible during active generation
The studio section header shows the "No API key — demo mode" banner based on !hasApiKey(). This check is not deferred during generation, so the yellow warning banner is distractingly visible while the user is already waiting for output.
Line 1413 Impact: Distracting during loading UX
M8
Gallery "Refresh Gallery" button active while dbLoading is true
The refresh button renders whenever activeFilter !== 'mine' without checking dbLoading. Clicking it while a load is in progress triggers a second concurrent loadPublicGallery() call, creating a race condition between two setState calls.
Lines 1905–1907 Impact: Gallery state race condition

Section 07
🧠 Prompt Engine Deep Audit — EXCELLENT

The 7-layer buildChibiPrompt function is the strongest part of the codebase. Each layer independently contributes structured prompt content aligned with the premium chibi specification.

1
BASE_SPEC — Core anatomy definition
Oversized expressive head, large luminous anime eyes with rich iris gradients and glossy reflections, soft rounded cheeks, gentle blush, small nose, warm smile, compact elegant body, clean confident linework, smooth gradient shading.
✓ Always present
2
styleDesc — Per-style rendering description
6 distinct render descriptions: Polished Anime (cel-shaded gradients, glossy eyes), Soft Watercolor (translucent washes, dreamy edges), Pixel Art (16-bit crisp sprites), 3D Render (Pixar-adjacent warmth), Sticker Style (bold outlines, flat fills), Pastel Sketch (hand-drawn pencil texture).
✓ All 6 styles distinct and detailed
3
tone — Tone bucket auto-detection
5 buckets: cute-professional, cute-artisanal, cute-luxury-lifestyle, cute-delicate-pastel, cute-cinematic-warm. Detected via keyword regex from user text or taken directly from profession preset's tone field.
✓ 5 buckets with distinct guidance text
4
roleBlock — Outfit + Props + Mood
When a profession is selected, injects outfit, props/details, and character mood as discrete sentences. Falls back to empty string for custom descriptions with no preset.
✓ Profession-aware, well structured
5
sceneBlock — Scene / Background
If profession has a scene, uses that (e.g., "bright modern hospital room with soft monitors"). Otherwise falls back to the selected background's sceneDesc (8 backgrounds each with ~30-word scene descriptions).
✓ 8 scene-aware backgrounds
6
compGuide — Composition framing
4 composition types: full-body-poster, seated-desk, lifestyle-portrait, paired-scene. Auto-selected based on tone bucket or overridden by profession preset's composition field.
✓ 4 composition types with framing instructions
7
finishParts — Softness / Sparkle / Negative constraints
Threshold-based: softness >70 = "dreamy rendering with gentle edge glow"; sparkle >70 = "surrounded by delicate magical sparkles." Appends negative constraints: no text, no watermarks, no logos, single character only, social-media ready.
✓ Responsive to slider values
⚠️ One Gap Found:

toneGuide is computed in Layer 3 but assembled into the prompt after sceneBlock (Layer 5) and compositionGuide (Layer 6). For best DALL-E 3 prompt weighting, tone guidance should appear earlier — ideally between Layer 3 (style desc) and Layer 4 (role block). This is a minor optimisation, not a bug.

Section 09
🛠️ Recommended Fixes — Priority Order
🔴 Do These First — Critical Functionality
H1
Fix Photo-to-Chibi — Add a clear disclaimer explaining DALL-E 3 cannot read images, or upgrade to GPT-4o Vision / gpt-image-1 which accepts image inputs. Do not silently ignore the uploaded photo.
H2
Fix HD Download size — Change HD generation to use size: '1792x1024' (landscape) or '1024x1792' (portrait) rather than just changing the quality flag.
M2+M3
Remove false tech claims — Update UX Section to say "OpenAI DALL-E 3" only. Rename Architecture section to "Roadmap" or clearly label unimplemented components as "planned."
🟠 Do These Next — High Impact UX
L4
Move GalleryCard outside GallerySection — Define GalleryCard as a top-level function to prevent unnecessary re-mounts on every gallery state change.
M1
Simplify action bar — Group into Primary (⬇️ PNG, 🖼️ Save, 🔗 Share) and a "More…" dropdown for HD, Copy, Meta export, Zoom. Hide "Add Key" inside the dropdown.
L7
Fix btoa() for Unicode passwords — Wrap with btoa(encodeURIComponent(password)) and match on login side to prevent crash for non-Latin character passwords.
M6
Fix metadata export memory leak — Add URL.revokeObjectURL(a.href) after the click in handleExportMeta.
H5
Fix nav highlight at bottom of page — Remove 'architecture' from the section tracker array since no nav link points to it. Only observe IDs that correspond to navbar links.
M8
Disable Refresh Gallery button while loading — Add disabled={dbLoading} to the refresh button to prevent double-trigger race conditions.
🔵 Polish Pass
L10
Add disabled styling to social login buttons or replace with a single "More sign-in options coming soon" note.
L9
Wire "Forgot password?" to an addToast('Password reset — coming soon! 🔐', 'info') or remove the link.
M4
Align footer brand description with hero copy: "Create premium chibi portraits, social-ready and identity-preserving."
M5
Add tooltip or sub-label to quota CTA: "3 free without signing in — 5/day with a free account."
M7
Hide the "No API key" banner while generating === true to reduce distraction during the 8–20s wait.
🔑 Prompt Engine Optimisation (Optional)

Move toneGuide earlier in the assembled prompt — between Layer 2 (style) and Layer 4 (role) — for improved DALL-E 3 prompt token weighting. DALL-E 3 weights earlier sentences more heavily.
